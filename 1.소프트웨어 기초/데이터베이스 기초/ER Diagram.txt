ER Diagram
  개체 : 사각형으로 표현
  속성 : 타원으로 표현
   ㄴ> 개체나 관계가 갖고 있는 고유 특성
     속성의 분류
	단일 값 속성 : 값을 하나만 갖는 속성 ex) 이름, 적립금 1명 당 하나의 값
	다중 값 속성 : 값을 여러개 갖는 속성 ex) 연락처, 저자 ) 여러명 존재 가능
		ㄴ> 다중 값 속성은 이중 타원으로 표현

	단순 속성 : 의미를 분해 할 수 없는 속성 ) 의미 1개 ex) 책 이름, 가격
	복합 속성 : 의미 분해 가능 속성 ) 의미 여러개 ex) 주소, 생년월일 등
			년, 월, 일로 분해가능 > 단순 속성이 모여 복합 속성 생성
	유도 속성 : 다른 속성 값에 의해 결정되는 속성
	     저장 속성: 실제 값 저장
	     유도 속성 : 필요 할 때 마다 값이 계산 됨) 저장할 필요 X
		ex) 책 가격과 할인율(저장 속성) -> 판매 가격이 할인율에 따라 바뀜
			유도 속성은 점선 타원으로 표헌
	널 속성 : NULL값의 허용되는 속성(결정되지 않은 값) / 공백. 0 과 의미가 다름
03.20
	키 속성 : 인스턴스를 식별하는데 사용 / 제약조건(개체 인스턴스 식별 필요) 밑줄로 표현
  
관계 : 마름모 모양으로 표현
   ㄴ> 개체 집합들 사이의 대응 관계(매핑)
     관계의 유형
	이항 관계 / 삼항 관계 / 순환 관계(하나가 자기 자신과)
     매핑 카디널리티(인스턴스가 맺고 있는 상대 개체 집합의 인스턴스 개수)
	일대일(1:1) 관계 : 개체A와 개체B의 각각 1개의 인스턴스와 관계를 맺을 수 있음 ex) 부부관계
	일대다(1:n) 관계 : 개체A 1개의 인스턴스에서 개체B의 다수의 인스턴스와 관계를 맺을 수 있음 ex) 소속 관계
	다대다(n:m) 관계 : 개체A와 개체B 서로 다수의 인스턴스와 관계를 맺을 수 있음 ex) 고객-책 관계

     관계의 참여 특성
	필수적 참여(전체 참여)
		모든 개체 인스턴스가 관계에 반드시 참여 ex) 고객 개체가 책 개체와 구매관계에 필수적 참여
			- 모든 고객은 책을 반드시 구매 / 이중선으로 표현
	선택적 참여(부분 참여)
		개체 인스턴스 중 일부만 관계에 참여 ex) 고객이 구매하지 않은 책이 존재할 수 있음

     관계의 종속성
	존재 종속 : 개체B가 개체 A에 종속되면 A가 삭제되면 B도 삭제됨을 의미
		의존적인 개체(약한 개체) : B / 강한 개체 : A
	특징 : 일반적으로 일대다의 관계 / 약한 개체는 강한 개체와 관계에 필수적 참여
		약한 개체를 구별해주는 속성 : 구별자 or 부분키
	   약한 개체는 이중 사각형 / 강한 개체와 맺는 관계는 이중 마름모
		약한 개체는 필수적 참여이기 때문에 이중선으로 연결
